## 产品定义

### 产品概述

AIR_Memory 是一个为 AI Agent 设计的本地部署的记忆系统, AI Agent 可以通过 AIR_Memory 高效的存储记忆和查询相关记忆.

### 功能定义

- AIR_Memory 应支持 macOS 和 Windows 操作系统的本地一键部署;
- AIR_Memory 应提供完整的部署手册和用户手册供人类阅读;
- AIR_Memory 在默认情况下应自启动;
- AIR_Memory 向 AI Agent 提供接口, AI Agent 可以通过这个接口保存记忆;
- AIR_Memory 向 AI Agent 提供接口, AI Agent 可以通过这个接口查询自己所需的相关记忆;
- AIR_Memory 在保存记忆时的操作时间不应大于 100ms;
- AIR_Memory 在查询记忆时的操作时间不应大于 100ms（快速查询模式）;
- AIR_Memory 应向人类提供 UI 接口实现如下功能:
  - 查询记忆数据;
  - 删除指定的记忆数据;
  - 查看 AI 保存记忆的记录, 包括: 保存记忆的时间/原始内容;
  - 查看 AI 查询记忆的记录, 包括: 查询记忆的时间/查询条件和返回的查询结果;
  - 查看每个记忆的价值反馈记录, 包括: 每次评价的时间/评价结果（有价值/无价值）;
  - 查看每个记忆的当前综合价值评分;

### 性能需求

- AIR_Memory 系统整体内存占用不应超过 8GB;
- AIR_Memory 系统磁盘占用不应超过 40GB；在存储空间触及上限之前，系统应自动删除低价值记忆中最旧的数据;
- AIR_Memory 向 AI Agent 提供记忆价值反馈接口, AI Agent 可以通过这个接口对查询结果中的每条记忆评价其是否有价值;
- AIR_Memory 应根据记忆的价值评分实现分级存储：将价值最高的记忆存储在高速存储层（支持快速查询），其余记忆存储在普通存储层（查询效率相对较低）；系统需在 8GB 内存上限约束下，最大化高速存储层的覆盖容量;
- AIR_Memory 向 AI Agent 提供的记忆查询接口中，应支持通过参数指定查询模式：
  - **快速模式**（`fast_only=true`）：仅在高速存储层中检索，响应时间不超过 100ms;
  - **深度模式**（`fast_only=false`，默认）：同时检索高速存储层和普通存储层，返回更全面的结果，响应时间可超过 100ms;
