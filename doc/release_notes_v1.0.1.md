## 修复问题

### Windows start.bat 在中文系统（GBK 代码页）下启动失败

**问题描述**

在 Windows 中文系统（代码页 CP936/GBK）中执行 `start.bat` 时，部分 REM 注释内容及脚本中的 URL 被当作命令执行，导致启动流程中断失败。

**根因分析**

v1.0.0 的 `start.bat` 使用 LF 行尾（Unix 风格）。Windows CMD 在 GBK 代码页模式下解析脚本时，UTF-8 编码中文字符的末字节（范围 0x81–0xFE）恰好落在 GBK 前导字节范围内，导致 GBK 解析器将 LF 行尾字节消费为汉字编码的一部分，行边界丢失，多行内容被合并解析为单条命令，进而引发执行错误。

**修复方案**

本次 Hotfix 通过以下三项变更彻底解决上述问题：

1. `start.bat` 第 2 行新增 `chcp 65001 >nul 2>&1`，在脚本执行前将代码页切换为 UTF-8（65001），确保后续内容均以 UTF-8 方式解析；
2. `start.bat` 行尾从 LF 改为 CRLF（Windows 标准行尾），避免行边界被 GBK 解析器错误消费；
3. 新增 `.gitattributes` 文件，锁定 `.bat` 文件始终使用 CRLF 行尾，防止后续 Git 操作（如跨平台 checkout）自动转换行尾导致问题复现。

## 升级说明

从 v1.0.0 升级至 v1.0.1 只需拉取最新代码，无需重新构建镜像或迁移数据。

**升级步骤**

1. 停止当前运行的服务：

```bat
docker compose stop
```

2. 拉取最新代码：

```bat
git pull
```

3. 重新启动服务：

```bat
start.bat
```

> **说明**：本次 Hotfix 仅修改启动脚本和 `.gitattributes`，后端/前端镜像无需重新构建，现有数据完整保留，升级过程对系统数据无任何影响。
