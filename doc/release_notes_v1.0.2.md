## 概述

本次发布为 v1.0.1 的 Hotfix 版本, 仅修复 Windows 平台启动脚本中全角中文符号在特定系统配置下引发的 CMD 解析异常问题, 不引入任何新功能或架构变更.

## 修复问题

### Windows start.bat 在部分 Windows 系统下全角符号导致 CMD 解析异常

**问题描述**

v1.0.1 的 `start.bat` 在部分 Windows 系统（主要是 Windows 7 或特定 Windows 10 配置）下执行时, 部分 REM 注释内容和 `echo` 命令内容被当作命令执行, 出现类似以下错误:

- `'或在命令提示符中执行' is not recognized as an internal or external command`
- `'下载地址：https:' is not recognized as an internal or external command`

**根因分析**

`start.bat` 中包含全角中文符号（`：`、`，`、`（`、`）`、`。`、`！`）, 其 UTF-8 编码字节序形如 `EF BC xx`, 其中第三字节（如 `0xBA`、`0xBF`、`0x88`、`0x89`、`0x82`、`0x81` 等）落在 GBK 前导字节范围（0x81-0xFE）内. 在部分 Windows 系统的 CMD 中, `chcp 65001` 的代码页切换未能完全生效, CMD 仍以 GBK 方式解析部分字节, 导致字节边界错位, 相关行内容被当作命令执行.

此外, 上述全角中文符号违反 `ai_rules/README.md` 中"禁止使用全角中文符号"的规范要求.

**修复方案**

将 `start.bat` 中所有全角中文符号替换为 ASCII 半角等价符号:

| 全角符号 | 替换为 | 说明 |
| --- | --- | --- |
| `：` (U+FF1A) | `:` | 全角冒号 → 半角冒号 |
| `，` (U+FF0C) | `,` | 全角逗号 → 半角逗号 |
| `（` (U+FF08) | `(` | 全角左括号 → 半角左括号 |
| `）` (U+FF09) | `)` | 全角右括号 → 半角右括号 |
| `。` (U+3002) | `.` | 全角句号 → 半角句号 |
| `！` (U+FF01) | `!` | 全角感叹号 → 半角感叹号 |

中文文字内容保留不变, `chcp 65001` 和 CRLF 行尾保留不变.

## 升级说明

从 v1.0.1 升级至 v1.0.2 只需拉取最新代码, 无需重新构建镜像或迁移数据.

**升级步骤**

1. 停止当前运行的服务:

```bat
docker compose stop
```

2. 拉取最新代码:

```bat
git pull
```

3. 重新启动服务:

```bat
start.bat
```

> **说明**: 本次 Hotfix 仅修改 `start.bat` 中的符号, 后端/前端镜像无需重新构建, 现有数据完整保留, 升级过程对系统数据无任何影响.
