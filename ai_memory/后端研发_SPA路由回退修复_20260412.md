# 后端研发 - SPA 路由回退修复

**标签**: `后端`, `main.py`, `SPA`, `Vue Router`, `FastAPI`, `exception_handler`, `StaticFiles`, `404回退`

## 任务背景

`StaticFiles(html=True)` 在 Starlette 1.x 不支持真正的 SPA 路由回退：
用户直接访问 Vue Router history 模式子路由（如 `/memories`）会得到 404，而非 `index.html`。

### 3. 修改 `backend/src/air_memory/main.py`

**新增导入**：
```python
from fastapi.responses import FileResponse, JSONResponse
from starlette.exceptions import HTTPException as StarletteHTTPException
```

**移除误导性注释**（`html=True` 不提供 SPA 回退）。

**注册异常处理器**（在 `StaticFiles` 挂载之后）：
```python
@app.exception_handler(StarletteHTTPException)
async def spa_fallback_handler(request, exc):
    if exc.status_code == 404:
        path = request.url.path
        if not path.startswith("/api/") and not path.startswith("/mcp"):
            if STATIC_DIR and os.path.isdir(STATIC_DIR):
                index_path = os.path.join(STATIC_DIR, "index.html")
                if os.path.isfile(index_path):
                    return FileResponse(index_path)
    return JSONResponse(status_code=exc.status_code, content={"detail": exc.detail})
```

## 关键实现要点

1. **处理器注册位置**：`exception_handler` 可在 `StaticFiles` 挂载之后注册，FastAPI 会正确应用
2. **路径判断顺序**：先检查 `/api/` 前缀，再检查 `/mcp` 前缀，均满足时才执行 SPA 回退
3. **文件存在检查**：双重检查 `STATIC_DIR` 目录和 `index.html` 文件是否存在，避免运行时错误
4. **Sparrow 原有 6 个测试全部继续通过**：`test_static_dir_unknown_route_returns_404` 使用不含处理器的独立 test_app，行为不受影响
