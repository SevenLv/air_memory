## 内容

### 任务背景

Issue 要求放弃 Docker 部署方案，改用依赖性更少的方式部署和运行系统。

### 架构决策

**新部署方案（Python 本机直接运行）**：
1. FastAPI（uvicorn）单进程，统一端口 8080 提供：REST API（/api/v1/）、MCP 服务（/mcp）、前端静态文件（/）
2. 前端 Vue.js 3 预构建 `frontend/dist/` 随仓库分发，用户无需 Node.js
3. 静态文件通过 `fastapi.staticfiles.StaticFiles(html=True)` 挂载
4. SPA 路由回退：自定义 `@app.exception_handler(StarletteHTTPException)` 处理 404，非 API/MCP 路径返回 `index.html`（注意：`StaticFiles(html=True)` 本身不支持真正的 SPA 回退，必须使用自定义异常处理器）
5. 自启动：macOS LaunchAgent（`~/Library/LaunchAgents/com.air-memory.plist`）/ Windows Task Scheduler
6. 数据存储于 `./data/`（相对项目根目录）

### 关键技术坑

**SPA 路由回退**：Starlette 1.x 的 `StaticFiles(html=True)` **不支持** SPA 路由回退，对不存在的路径返回 404 而非 index.html。必须通过 `@app.exception_handler(StarletteHTTPException)` 实现自定义 404 处理器才能支持 Vue Router history 模式。
