# AIR_Memory v1.2.1 发版管理记忆

## 发版背景

Issue #30 (Windows 上浏览器 Edge 切换到别的程序时不断激活自身) 在 v1.2.0 应用
--no-access-log 后仍有问题. 深度分析发现真正根因是 FastMCP 模块导入时通过
logging.basicConfig() 将 root logger 设为 INFO+RichHandler, 导致 MCP 工具调用日志
写入 stderr, 引发 Windows FlashWindowEx 任务栏闪烁.

## 修复内容

- backend/src/air_memory/main.py: 在所有 import 后添加
  logging.getLogger().setLevel(logging.WARNING) 重置 root logger 级别
- 版本号: 1.2.0 -> 1.2.1 (main.py APP_VERSION, backend/pyproject.toml, start.bat, start.sh banner)

## 发版流程

1. 创建 doc/release_notes_v1.2.1.md
2. 更新 README.md 版本号至 v1.2.1
3. 更新 .github/workflows/release.yml 指向 release_notes_v1.2.1.md
4. 合并 PR 后通过 workflow_dispatch 触发 release.yml, 输入 tag=v1.2.1
