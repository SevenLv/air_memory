## 关键实现细节

### 乱码检测逻辑 (isGarbled)
- 纯 ASCII 内容不视为乱码（先判断是否含 charCode > 127 的字符）
- 含非 ASCII 字符时，若问号占比 > 30% 则判断为乱码
- 乱码时显示 `<el-tag type="warning">乱码</el-tag>` + tooltip 说明 + 灰色斜体文本

### getAllFeedbackLogs API
- 路径: `GET /api/v1/logs/feedback`
- 参数映射: `memoryId` → `memory_id`, `pageSize` → `page_size`, `startTime` → `start_time`, `endTime` → `end_time`
- 返回类型: `FeedbackLogsWithTotalResponse` (`logs`, `count`, `total`)

### FeedbackView 重构要点
- onMounted 时自动加载第一页（无需用户操作即可看到数据）
- 日期时间选择器: `value-format="YYYY-MM-DDTHH:mm:ss"` 对应后端 ISO 格式
- Promise.all 同时发起日志查询和（有 memoryId 时的）价值评分查询
- getValueScore 失败时 `.catch(() => {})` 静默处理，不影响日志列表
- 移除了 ElMessage import（noUnusedLocals: true 会报错）

### 测试更新要点
- Mock 改为 `getAllFeedbackLogs`（替换旧的 `getFeedbackLogs`）
- 标题断言改为检查 '查询条件' 和 '反馈记录列表'
- "ID 为空时" 测试: 验证 `getValueScore` 不被调用（而非 `getAllFeedbackLogs`）
- "未找到" 测试: 改为验证失败时不显示价值评分面板
