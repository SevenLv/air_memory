## Issue #42 技术分析

根因：v1.2.3 仅做了文档更新，未同步更新代码中的版本号。
`APP_VERSION = "1.2.2"` 、`backend/pyproject.toml` version、start.bat/start.sh banner 均未更新。
前端通过 `GET /api/v1/version` 动态获取版本号，后端返回 "1.2.2"，因此 UI 显示 1.2.2。

修复：将所有版本号更新为 "1.2.4"（同时包含其他 3 个 Issue 的修复）。

## Issue #37 技术分析

根因：v1.2.0 时期 PYTHONUTF8=1 未在所有场景生效，Windows CP1252 代码页下
中文字符通过 locale.getpreferredencoding() 编码为 CP1252 导致问号损坏。
历史数据已永久损坏，无法恢复（原始内容为 ????）。

现状：PYTHONUTF8=1 已在 start.bat/start.sh 中设置，新数据应正常存储。
UI 层对乱码内容无任何提示，用户体验差。

修复：
1. 后端 log/service.py: 添加防御性问号比例检测，疑似乱码时输出 WARNING 日志
2. 前端 LogsView.vue: 添加 isGarbled() 函数检测（非 ASCII 内容中问号占比 > 30%），
   显示"乱码"badge + tooltip（说明这是历史遗留，新记忆不受影响）

## Issue #30 技术分析

现有修复（已在 main.py）：
- --no-access-log（start 脚本中）
- logging.getLogger().setLevel(logging.WARNING)（抑制 FastMCP INFO 日志）

残留原因：chromadb、sentence_transformers 等第三方库可能有自己独立的
logger handler，不经过 root logger，仍向 stderr 输出 INFO 日志，
引发 Windows CMD 窗口 FlashWindowEx 闪烁。

追加修复：在 main.py 中对已知噪声库显式设置 WARNING 级别：
- chromadb, chromadb.api, chromadb.db
- sentence_transformers
- transformers, transformers.tokenization_utils_base
- httpx, httpcore, h11
- mcp, mcp.server, mcp.server.lowlevel, mcp.server.streamable_http,
  mcp.server.streamable_http_manager
- asyncio

## Issue #43 技术方案

### 后端 API

新端点：GET /api/v1/logs/feedback
参数：page, page_size, memory_id（可选）, start_time（可选）, end_time（可选）
响应：FeedbackLogsWithTotalResponse { logs: FeedbackLog[], count: int, total: int }
实现：FeedbackService.get_all_feedback_logs()，使用 WHERE 子句动态拼接

### 前端 UI

重构 FeedbackView.vue：
- 查询条件面板：记忆 ID 输入 + 日期范围选择器（el-date-picker datetimerange）
- 综合价值评分面板：仅当指定记忆 ID 时显示（复用现有功能）
- 反馈记录列表：分页表格（el-pagination），页面加载时自动获取第一页
- 整合现有查询功能（按记忆 ID 查看价值评分）
