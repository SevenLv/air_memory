# 前端研发 - 记忆管理 UI 分页查询详情页

## 标签

前端, 记忆管理, 分页, 时间段查询, ID查询, 详情页, 路由, 用户手册, v1.2.7

## 任务概述

实现记忆管理 UI 新需求:
- `/memories` 默认展示最近记忆;
- 每页 20 条分页;
- 支持按记忆 ID 和时间段筛选;
- 列表项点击进入详情页;
- 详情页展示 ID, 原始数据, 提交时间, 价值评分.

## 最小改动路径

1. 复用现有 `GET /api/v1/logs/save` 作为记忆列表数据源.
2. 将 `MemoriesView.vue` 从语义查询改为管理列表:
   - 查询条件: 记忆 ID + 时间范围;
   - 默认加载最近记录;
   - 前端分页固定 20 条.
3. 新增详情页 `MemoryDetailView.vue` 和路由 `/memories/:memoryId`.
4. 新增后端接口 `GET /api/v1/logs/save/{memory_id}` 供详情页按 ID 精确查询.
5. 详情页价值评分复用 `GET /api/v1/memories/{id}/value-score`.
6. 列表跳转详情时通过路由 `state` 透传当前行, 详情页优先使用该状态减少重复请求.
7. 更新 `MemoriesView.spec.ts` 并新增 `MemoryDetailView.spec.ts`.
8. 更新 `doc/user_guide.md` 的 `/memories` 使用说明.
