# 项目协作 - charset 中文乱码修复与发版准备

## 标签

charset, utf-8, 中文乱码, REST API, 中间件, v1.2.12, 发版, Issue

## 任务背景

Issue: AI 不能保证每次调用接口都设置 charset, 导致记忆信息中的中文乱码。

## 执行路径

1. 安排 Lydia 处理技术工作 (架构分析 + Neo 实现 + Sparrow 测试 + 技术文档)
2. Nia 更新用户手册 (doc/user_guide.md v1.9)
3. 安排 Wii 执行验证
4. Nia 执行发版准备

## 技术修复要点 (Lydia 完成)

- 在 main.py 新增 _ForceUTF8JSONMiddleware 纯 ASGI 中间件
- 中间件对所有 application/json 请求强制设置 charset=utf-8
- 覆盖场景: 无 charset / 错误 charset / 正确 charset 均不受影响
- 版本升级: 1.2.11 -> 1.2.12
- 新增 sad_v1.13.md, release_notes_v1.2.12.md

## 用户手册更新要点 (Nia 完成)

- user_guide.md v1.9: 移除强制设置 charset 要求, 改为说明服务端自动处理

## 验证安排

- 安排 Wii 按用户手册验证功能, 重点验证: 无 charset 的 REST API 调用中文内容完整性

## 发版文件

- doc/release_notes_v1.2.12.md
- doc/sad_v1.13.md
