## 标签

## 内容

### 子模块初始化

- `copilot-setup-steps.yml` 已配置 `git submodule update --init --recursive`，环境启动时会自动初始化；
- 若启动后子模块目录为空（`git submodule status` 前缀为 `-`），应手动执行 `git submodule update --init --recursive`；
- 初始化后必须读取 `/ai_rules/README.md` 及各分类规则文件；

### 必须遵守的规则（ai_rules v0.1）

引用文件：
- `/ai_rules/README.md` (v0.1)
- `/ai_rules/memory_rules/basic_rules.md` (v0.1)
- `/ai_rules/task_rules/basic_rules.md` (v0.1)
- `/ai_rules/file_rules/markdown_rules.md` (v0.1)
- `/ai_rules/proj_rules/basic_rules.md` (v0.1)

规则摘要：

1. **记忆规则**：每次任务前读取 `/ai_memory` 中相关记忆文件；任务完成后在 `/ai_memory` 中保存记忆（文件名格式：`[主题]_[简要描述]_[yyyyMMdd].md`，使用中文文件名）；
2. **任务规则**：读取所有规则后，提炼适用当前任务的简明规则并记录为记忆；
3. **Markdown 规则**：遵守 markdownlint 规则；工程根目录无配置文件时，使用 `/ai_rules/file_rules/.markdownlint.jsonc`（关闭 MD013、MD033）；
4. **工程规则**：工程自身规则与 ai_rules 冲突时，以工程规则为准；
5. **基本要求**：必须使用中文（沟通、文档、代码注释）；禁止全角中文符号；

### 文档变更记录规范

- 首版文档（v1.0）变更记录内容只能填写 "初稿"；
- 文档修订（非版本升级的编辑）不改变变更记录描述；
- 需要创建新版本时：复制原文件到新版本文件名，修改新文件，并在变更记录表中追加新版本行；
