# 架构分析_Issue28地址问题与Issue27版本显示修复方案_20260413

## 标签

Issue#28, Issue#27, start.bat, start.sh, uvicorn, 0.0.0.0, localhost, 127.0.0.1, 版本显示, v1.2.0, App.vue, /api/v1/version, main.py, pyproject.toml

## 问题根因分析

### Issue #28: 地址显示 http://0.0.0.0:8080

**根因**: start.bat/start.sh 中 echo 语句已正确输出 `http://localhost:PORT`, 但 uvicorn 以 `--host 0.0.0.0` 启动后会自行打印:

```
INFO:     Uvicorn running on http://0.0.0.0:8080 (Press CTRL+C to quit)
```

用户看到此 uvicorn 消息后尝试访问 http://0.0.0.0:8080, 导致无法访问.

**修复方案**: 将 `--host 0.0.0.0` 改为 `--host 127.0.0.1`, uvicorn 将打印 `http://127.0.0.1:8080`, 该地址浏览器可直接访问.

### Issue #27: UI 和启动脚本未显示版本号

**根因**: 系统各处均未定义或展示版本号:
- backend/pyproject.toml: version = "1.0.0" (已过时)
- backend/src/air_memory/main.py: FastAPI(version="1.0.0") (已过时, 无版本 API)
- frontend/src/App.vue: 仅显示标题, 无版本
- start.bat/start.sh: 启动 banner 无版本号

## 修复方案（v1.2.0）

### 需修改的文件清单

| 文件 | 变更内容 |
| --- | --- |
| `start.bat` | `--host 0.0.0.0` 改为 `--host 127.0.0.1`; banner 添加 v1.2.0 |
| `start.sh` | `--host 0.0.0.0` 改为 `--host 127.0.0.1`; banner 添加 v1.2.0 |
| `backend/pyproject.toml` | version 更新为 "1.2.0" |
| `backend/src/air_memory/main.py` | 新增 APP_VERSION 常量 "1.2.0"; 更新 FastAPI version; 新增 /api/v1/version 端点 |
| `frontend/src/api/index.ts` | 新增 getVersion() 函数 |
| `frontend/src/api/types.ts` | 新增 AppVersion 接口 |
| `frontend/src/App.vue` | 请求版本 API, 在 header 中显示版本号 |
| `frontend/package.json` | version 更新为 "1.2.0" |

### 版本 API 设计

在 main.py 中直接定义 (与 /health 端点保持一致, 位于 /api/v1/version 路径):

```python
APP_VERSION = "1.2.0"

@app.get("/api/v1/version", tags=["system"])
async def get_version() -> dict:
    """获取系统版本号。"""
    return {"version": APP_VERSION}
```

### 前端版本显示设计

App.vue header 中在标题右侧/下方展示灰色小字版本号, 通过 API 动态获取.
